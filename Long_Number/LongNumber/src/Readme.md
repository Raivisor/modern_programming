# Реализация LongNumber.cpp

## Ключевые архитектурные решения:

1. **Обратный порядок хранения цифр**
   - Индекс 0 — единицы, 1 — десятки, и т.д.
   - Упрощает сложение/вычитание (перенос идёт вперёд по массиву)
   - Упрощает умножение (индексы произведений не сдвигаются)
    - P.S. я потом постоянно забывал, что зачастую придется идти от конца массива к началу, поэтому часто в циклах вместо i-- встречался i--. Но как по мне, это все же упростило разработку арифметических операторов

2. **Нормализация нуля**
   - Во всех конструкторах и операторах присваивания после создания вызывается `normalize()`
   - Любой ноль (включая "-0") становится положительным
   - Пояснение: Было решено не вводить отдельное значение для знака нуля, т.к. это бы нагружало проверку знака(с 2 работать легче, чем с 3). 0 был присвоен положительный знак. Чтобы не возникало поломок, был написан отдельный метод нормализации

3. **Статические вспомогательные методы**
   - Не привязаны к конкретному объекту
   - Работают только с модулями, знак обрабатывается отдельно
   - Позволяют избежать дублирования кода
   - Пояснение: логика реализации арифметических операций сокрыта от пользователей

## Сложные моменты:

- **add_abs**: выделяем память с запасом (max_length + 1), потом уменьшаем length если нет переноса
- **subtract_abs**: обрабатываем заём (borrow)
- **divide_abs**: на каждом шаге умножаем остаток на 10 и добавляем следующую цифру
